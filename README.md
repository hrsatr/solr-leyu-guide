# solr-leyu-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定的域名或具体网站，而是作为一组通用的前端资源或信息展示页面进行存储和维护。

## 项目简介

`sorl-leyu-guide` 是一个轻量级的静态页面集合仓库。每个页面都独立存在，不依赖复杂的后端服务或构建工具。仓库的主要目的是提供一个集中、可追溯的版本管理环境，方便查阅、分享以及协作更新这些 HTML 文件。

## 目录说明

```
solr-leyu-guide/
├── index.html          # 入口页面或导航页面
├── page1.html          # 示例页面一
├── page2.html          # 示例页面二
└── assets/             # 公共资源文件夹（样式、脚本、图片等）
    ├── style.css
    ├── script.js
    └── images/
```

- `index.html`：可选的入口页面，用于快速索引或跳转。
- 其他 `.html` 文件：各自独立的页面，彼此之间没有强制依赖关系。
- `assets/`：存放页面公用的 CSS、JavaScript 和图片等资源文件。

## 页面归档说明

所有 HTML 页面遵循以下通用原则：

- 页面文件均为纯前端实现，可直接在浏览器中打开运行。
- 不包含任何服务器端逻辑或动态数据请求。
- 页面之间的链接、样式引用均使用相对路径。
- 每个页面保持简洁、可读，便于直接查看或二次修改。

## 维护说明

- 如需新增页面，请直接在仓库根目录或合适子目录下创建新的 `.html` 文件。
- 修改已有页面时，建议保持原有的文件结构和命名风格。
- 若涉及公共资源（如样式或脚本）的变更，请同步更新所有引用该资源的页面。
- 提交前请确保页面在主流浏览器中能正常渲染，无明显错误。
